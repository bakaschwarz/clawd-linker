5. Add a new exported function `cleanEmptyDirs(removedPaths, projectPath)`:
```javascript
export async function cleanEmptyDirs(removedPaths, projectPath) {
  const dirs = new Set();
  for (const p of removedPaths) {
    let dir = path.dirname(p);
    while (dir !== projectPath && dir.startsWith(projectPath)) {
      dirs.add(dir);
      dir = path.dirname(dir);
    }
  }
  const sorted = [...dirs].sort((a, b) => b.length - a.length);
  for (const dir of sorted) {
    try {
      const entries = await readdir(dir);
      if (entries.length === 0) await rmdir(dir);
    } catch {
      // ENOTEMPTY or already removed — skip
    }
  }
}
```

This function collects all parent directories of removed symlinks up to (but not including) projectPath, sorts them deepest-first by path length, and removes only empty ones.

| Boundary | Description |

| CLI args -> manage command | User-supplied --dry-run and --yes flags (trusted — single-user CLI) |
| data.json -> filesystem ops | State file read used to determine uninstall targets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | data.json symlink paths used in unlink | accept | Single-user tool, data.json is local. Existing lstat guard in uninstallPackage already verifies target is a symlink before unlinking. |
| T-02-02 | Information Disclosure | dry-run output reveals file paths | accept | Local CLI tool, output goes to user's own terminal only. |
| T-02-03 | Denial of Service | cleanEmptyDirs could climb above projectPath | mitigate | Guard `dir !== projectPath && dir.startsWith(projectPath)` prevents climbing above project root. |
</threat_model>
